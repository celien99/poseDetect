# 座椅区域标定说明

本文档说明如何在当前项目中把座椅区域标定正确，以及三类框各自应该怎么画。

## 1. 为什么标定很重要

当前项目在固定机位方案下，动作识别强依赖 `seat_regions`。

也就是说：

- 座椅整体框错了
- 侧面区域框错了
- 底部区域框错了

后面的动作判断就会一起错。

因此在当前项目里，区域标定通常比“调模型”更优先。

## 2. 当前需要标定哪三个区域

当前项目需要这三个框：

- `overall`
- `side_surface`
- `bottom_surface`

它们的含义分别是：

### `overall`

表示整张座椅主体的大致范围。

用途：

- 作为整椅参考框
- 在启用座椅检测模型时，作为模板区域的参考基准

### `side_surface`

表示业务上定义的“应当触摸的侧面区域”。

用途：

- 判断 `touch_region`
- 判断手腕是否真正接触目标侧面，而不是碰到别的地方

### `bottom_surface`

表示业务上定义的“应当接触或抬起的底部区域”。

用途：

- 判断 `lift_region`
- 判断双手是否进入目标底部区域

## 3. 推荐标定方式

建议直接使用项目自带命令：

```bash
python -m seat_inspection calibrate-regions --source "mvs://0?timeout_ms=1000" --output configs/seat_regions.front.json
```

也可以对视频或图片标定：

```bash
python -m seat_inspection calibrate-regions --source "data/demo.mp4" --output configs/seat_regions.demo.json
python -m seat_inspection calibrate-regions --source "data/frame.jpg" --output configs/seat_regions.demo.json
```

程序会依次要求你框选：

1. `overall`
2. `side_surface`
3. `bottom_surface`

## 4. 标定前的准备

标定前建议先满足下面条件：

1. 相机机位已经固定
2. 分辨率已经固定
3. 座椅摆放位置基本固定
4. 选择一帧无遮挡、清晰、最能代表日常工况的画面

如果机位、焦距、画面裁切还在变化，那么先不要标。

## 5. 三类框到底怎么画

## `overall` 怎么画

目标：

- 框住整张座椅主体
- 尽量贴边
- 不要切掉椅体关键边缘

建议：

- 包含座椅的主要轮廓
- 不要把大量背景也包进去
- 不要只框中间一小块

经验：

- 宁可略微留一点边，也不要裁掉真实椅体边缘

## `side_surface` 怎么画

目标：

- 只框业务上允许/要求触摸的那一块侧面区域

建议：

- 按业务定义框，不按视觉随便框
- 如果动作只发生在局部侧面，就只框局部
- 不要把整个座椅侧边全部包进去，除非业务本身允许

经验：

- `side_surface` 画太大，容易误报
- `side_surface` 画太小，容易漏报

## `bottom_surface` 怎么画

目标：

- 只框业务上定义的底部接触/抬起区域

建议：

- 只包含真正需要接触或抬起的位置
- 不要把底部周围背景区域一起包进去
- 不要把“可能经过但不属于动作目标”的区域也算进去

经验：

- `bottom_surface` 太大，双手稍微靠近就容易误判
- `bottom_surface` 太小，动作明明做了也可能不触发

## 6. 一个简单的画框原则

如果你一开始不知道怎么画，可以先按这个原则：

- `overall`：偏大但贴近座椅主体
- `side_surface`：先略大一点，后续根据误报收紧
- `bottom_surface`：先覆盖真实动作区域，再根据误报缩小

也就是说：

- 第一版先保证别漏太多
- 第二版再慢慢减少误报

## 7. 标错之后会出现什么现象

### `overall` 标错

常见现象：

- 整个座椅框位置明显不对
- 如果后续启用 `seat_model_path`，子区域映射也会一起偏

### `side_surface` 标错

常见现象：

- 人手明明触摸侧面，但 `touch_side_surface` 不触发
- 或者人手只是从附近经过，也触发了触摸动作

### `bottom_surface` 标错

常见现象：

- 双手明明伸到底部了，但 `lift_seat_bottom` 不触发
- 或者双手根本没有做标准抬起，也被误判为成立

## 8. 标定完成后怎么验证

标完后不要立刻相信它，先做一次推理验证。

建议流程：

1. 把标定结果填入推理配置
2. 先关闭实时窗口
3. 打开可视化输出
4. 先检查视频中的框，再看动作结果

建议配置：

```json
"show_window": false,
"save_visualization": true
```

然后执行：

```bash
python -m seat_inspection infer --config your_config.json
```

优先看输出视频里的三个框：

- 绿色 `seat`
- 黄色 `side`
- 紫色 `bottom`

验证标准：

- `seat` 是否贴合整张座椅
- `side` 是否覆盖目标触摸区域
- `bottom` 是否覆盖目标底部区域

## 9. 如何判断问题是“区域错了”还是“规则错了”

可以按下面方式判断：

### 情况 A：框本身明显歪了

结论：

- 优先是区域标定问题

### 情况 B：框基本对，但动作没触发

结论：

- 先调规则参数

优先看：

- `hold_frames`
- `wrist_margin`
- `min_wrist_count`
- `lift_ratio_threshold`

### 情况 C：框对、规则也不严，但结果还乱

结论：

- 再去检查姿态关键点稳定性

## 10. 一次标定不准怎么办

这是正常的。

建议做法：

1. 先标一版
2. 输出标注视频
3. 看框是否贴合
4. 微调区域
5. 再跑一次

不要在没看可视化结果前反复盲改坐标。

## 11. 当前项目的一个重要结论

在固定机位场景下：

**正确的区域标定，往往比额外训练一个模型更重要。**

因为当前项目的动作判断核心是：

- 人体关键点
- 座椅区域
- 几何关系
- 状态机顺序

所以如果窗口里框都错了，优先修区域，不要先怀疑整条推理链。
